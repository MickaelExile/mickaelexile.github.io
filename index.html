Applicatif bureautique

"Jeu sous forme de QCM"

Lors de ma première année de BTS et en plein confinement, j'ai dû réaliser en totale autonomie un jeu sur application permettant de répondre à un questionnaire à choix multiples. En partant d'une maquette, j'ai tout d'abord réalisé une page connexion, puis une base de donnée, ainsi qu'un mode administrateur. La page connexion a été créée pour permettre de choisir le mode : être l'administrateur, ou être le joueur. Si le mode joueur est choisi, cela permet de répondre au QCM et de comptabiliser son score, puis de le sauvegarder. Le joueur est inséré dans la base de donnée, après création du login/mot de passe. Le mode administrateur permet, quant à lui, de créer des thèmes; des niveaux; des questions; des réponses; et enfin, de faire la différence entre les réponses correctes et fausses.

  • Client: Lycée Chevrollier
  • Catégorie: C#/SQL(Base de donnée)

Applicatif web

"Location de vélo"

Ce projet est réalisé avec une méthode MVC (Modèle-Vue-Contrôleur) en PHP. À partir d'une base de donnée, j'ai réalisé une page connexion pour: soit se connecter en tant qu'administrateur, soit adhérent. La base de données a été réalisée en SQL, sur la base d'un modèle relationnel créé en amont par moi-même. En mode administrateur, on peut: soit réparer un vélo, soit le mettre en disponible, soit le déposer à une borne (via une liste déroulante), soit le mettre en réparation. Via le mode adhérent, il y a deux possibilités: soit louer un vélo (en y précisant: le nom, le prénom, la date, l'heure de location), soit déposer un vélo (en y précisant: la date, l'heure de dépot et les points gps). Enfin, un bouton déconnexion a été créé pour pouvoir se déconnecter.

  • Client: Lycée Chevrollier
  • Catégorie:PHP/MVC/SQL(Base de donnée)
